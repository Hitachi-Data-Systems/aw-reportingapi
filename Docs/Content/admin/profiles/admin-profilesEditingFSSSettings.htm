Editing file sync and share settings in a profile

For each profile that has been created, you can adjust the users' file synchronization and sharing settings. The file synchronization and sharing settings include:

•How the users in the profile can share

•Who the users in the profile can share with

•How many and which HCP Anywhere applications the users in the profile can install

•The NAS shares that the users in the profile can access in HCP Anywhere

To edit the file synchronization and sharing settings for an existing profile:

1.In the top-level menu, mouse over the Access tab to display a secondary menu.

2.Click on Profiles.

3.In the list of profiles, click on the profile you want.

The profile expands.

4.In the profile, click on the File Sync and Share tab.

5.To enable or disable file synchronization and sharing, select or deselect the Enable file sync and share option, as applicable.

6.To configure file synchronization and sharing settings, make the changes you want on the Core, Devices, Sharing, Mobilization, and Data Exclusion panels:

Core

•Quota — In this field, enter the amount of space each user has for adding files to HCP Anywhere.

Devices

•Maximum number of devices —The number of HCP Anywhere applications that a user can have registered with their HCP Anywhere account at one time. You can enter any integer from 0 to 11. Entering 0 means that users in the profile can only access HCP Anywhere through the User Portal.

•Device types allowed — The types of HCP Anywhere applications that the users in the profile can register with HCP Anywhere. You can always disable one of the HCP Anywhere applications for a profile once the profile has been created. When you do this, the applications that have been disabled are disconnected from HCP Anywhere.

One of the devices types listed here is FSS API. FSS API refers to the File Sync and Share API. The File Sync and Share API is an HTTP API for integrating custom and third-party applications with HCP Anywhere.

Notes:  

•If the File Sync and Share API is enabled, all applications connected to a user's HCP Anywhere account through the File Sync and Share API count as a single device. For example, if a user has three applications connected to HCP Anywhere by the File Sync and Share API, those three applications collectively count as one device registered to the user's HCP Anywhere.

•If the File Sync and Share API is disabled, users cannot:

–Use applications you have integrated with HCP Anywhere using the File Sync and Share API to access their HCP Anywhere accounts.

–Integrate applications with HCP Anywhere themselves using the File Sync and Share API.

• For information on viewing the File Sync and Share API documentation, see Viewing HCP Anywhere documentation.

•Automatic Deregistration — In this section, you can configure deregistration settings for the HCP Anywhere applications of the users in the profile.

•Automatically deregister after N days of inactivity — The amount of time an HCP Anywhere application can be inactive before the device is disconnected from HCP Anywhere. You can enter any integer that is less than the Archive File Events setting. See Archiving file event records for information on that setting.

When an application is deregistered from HCP Anywhere, the HCP Anywhere server stops synchronizing files with the application.

•Delete all files when an inactive desktop device is deregistered by the system — When selected, the files in a user's desktop app folder are deleted from their computer if their desktop application is deregistered by the system. This deletes the files on the user's computer, not in HCP Anywhere.

By default, when an HCP Anywhere mobile application is deregistered, all the user's files are deleted from the application.

Note: You can also deregister the applications of individual users instead of deregistering applications on a profile level. See Deregistering user devices for information on doing this.

Authentication Token Expiration — When a user installs and registers an HCP Anywhere application on a device, the device is registered with the HCP Anywhere system and given an authentication token. When the token expires, the user needs to reenter his or her password to continue using the HCP Anywhere application.

•Desktop devices — The amount of time that the authentication tokens for the HCP Anywhere desktop application and Outlook Add-in remain active. You can choose any time between 1 day and 50 years. The default is 7 days.

•Mobile devices — The amount of time that the authentication tokens for the HCP Anywhere mobile application remain active. You can choose any time between 1 day and 50 years. The default is 7 days.

•FSS API — The amount of time that the authentication tokens for applications connected to HCP Anywhere by the File Sync and Share remain active. You can choose any time between 1 day and 50 years. The default is 7 days.

Note: Each application connected to HCP Anywhere through the File Sync and Share API gets its own authentication token and is not connected to the authentication token of a device.

Sharing

•Enable read link sharing — When selected, users in the profile are allowed to share links internally with read access. A user who receives a link with read access can view and download the file or folder that was shared.

•Enable public read link sharing — When selected, users in the profile can share links publicly with read access. If a link is shared publicly, it can be accessed by anyone, even those who are not HCP Anywhere users.

•Enable upload link sharing — When selected, users in the profile can share links internally with upload access. Upload access only applies to folders. A user who receives a link with upload access can upload files to the folder that was shared. However, if the link is not also set with read access, the user cannot view or download any of the files in the folder.

•Enable public upload link sharing — When selected, users can share links publicly with upload access. Upload access only applies to folders. A user who receives a link with upload access can upload files to the folder that was shared. If a link is shared publicly, it can be accessed by anyone, even those who are not HCP Anywhere users.

•Allow links without expiration — When selected, users can share links that never expire.

•Maximum days to share — The maximum amount of time that a link can remain active. You can enter any integer from 1 to 18,262 days (1 day through 50 years). If Allow links without expiration is enabled, this setting does not apply.

•Default days to share — The default amount of time that links remain active. A user can change this value when sharing a link.

Note: Link sharing can be enabled or disabled for the entire system. See Configuring link sharing for the system for information on doing this.

•Enable Team Folders — When selected, users in the profile can create and share team folders.

•Threshold for requests — The maximum amount of storage space permitted for each team folder. The user who creates a team folder sets the storage space for the folder. If a user sets the storage space of a team folder over this limit, a request for the additional space is sent to you for approval. For information on answering these requests, see Accepting and declining Team Folder storage requests.

Note: Unlike links, you can work with the team folders that users create. On the Team Folders page in the Management Console, you can view all the existing team folders in the system, answer team folder storage requests, invite users to manage team folders, and delete team folders. For information on doing these things, see Managing team folders.

•Limit folder and link sharing — When selected, members of the current profile can only initiate sharing with those who are part of one of the profiles in the Added Profiles section. This applies for both link sharing and folder sharing.

•Available Profiles— This table contains all the profiles in the HCP Anywhere system. Click on the plus sign ( ) to add a profile to the Allow Sharing with These Profiles section.

•Allow Sharing with These Profiles — This table contains the profiles that the current profile can share with when limit sharing is enabled.

Important:  

•Since users do not need to authenticate to access a public link, limiting sharing here does not prevent users from sending public links. If you want complete control over who your users can share with, enable limit sharing and disable public links.

•If you want to prevent users from sharing with anyone, enable limit sharing, make sure the Allow Sharing with These Profiles section does not contain any profiles, and disable public link sharing.

•Limit sharing is one directional. For example, if you limit sharing in profile A so that profile A cannot initiate sharing with profile B, profile B can still initiate sharing with profile A. To completely limit sharing between the two profiles, you need to limit sharing in both profiles.

Mobilization

In this section, you can configure the network shares that the users in the profile can access in their HCP Anywhere accounts. Each network share appears as a folder in HCP Anywhere.

Notes:  

•If your networks shares are not available on this page, it is because they have not been added to the HCP Anywhere system. See Working with NAS servers for information on doing this.

•If SAML authentication is enabled for the profile, the users in the profile cannot access any network shares.

To make a network share available to the users in the profile:

a.In the Server field, select a NAS server.

b.From the list of shares that appears, select the shares you want.

c.Click on the right arrow button (  ).

d.Click on Save Settings.

Data Exclusion

•Exclude these file extensions — When selected, users in the profile cannot upload files to HCP Anywhere with the extensions are in the file extension field.

•In the file extension field, enter the file extensions that you want users to be prevented from uploading to HCP Anywhere. For example, if you enter png into the file extensions field, users cannot upload files with the png extension.

Note: Do not include periods (".") in the list of file extensions. If a period is included, users are still able to upload files with the specified extension.

7.At the bottom of each panel, click on Save Settings.
