Requesting an initial report and paging

When running a report, the initial request returns up to the maximum number of entries specified in the maxResults input parameter of your request body. The default value of this parameter is 100 entries, but the value can be modified. If there are more pages to your report, then a 206 response code is returned in the response header and a pageToken is returned at the end of the response body. To request an additional page of your report, resend the same request, but replace the input parameters in the request body with the pageToken and, optionally, a maxResults value.

The following is a table showing the input parameters for the subsequent request body.

JSON Type Description
pageToken String A string used to request an additional page of an incomplete report.
maxResults Integer Optional. The maximum number of entries to return. Default is 100. The minimum value is 1. The maximum value is 5000.

The following example shows the request body parameters stored in a file named userReportParams.json. The request body contains the pageToken parameter returned in the report..

Request body

userReportParams.json

{
    "pageToken": "2d2e7474f44be4a0654"
}

The following example requests an additional page of the user file modifications audit report. Substitute myUniqueID with your device-id.

Request with curl command line

curl -kiX POST -T userReportParams.json -H "Accept: application/json" -H "Authorization: Bearer ya29.qwHWIcw1FLhJh4-Y84FP_mp-PqaTgS5vHCNENuAB42qQ_4C5QuO4tmvTFE-XeLbYtDFhDGN65dA6yg" -H "X-HCPAW-MANAGEMENT-API-VERSION: 2.1.0" -H "X-HCPAW-DEVICE-ID:myUniqueID" "https://api.anywhere.example.com:8000/mapi/report/audit/user/activity/file/modifications"
