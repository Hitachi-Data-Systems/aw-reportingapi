Example: Generating a report

Hereâ€™s a sample POST request that generates an audit report. In this example, the report returns all file modifications made by the user lgreen between the startTime Mon, 16 Nov 2015 16:18:02 GMT and endTime Mon, 16 Nov 2015 18:47:31 GMT. The times have been converted respectively to the Unix Timestamp Conversion (UTC) times 1447690682 and 1447699651. The request body for this example is stored in a file named userReportParams.json.

Creating a request body

Here is an example of the request body used for the audit report.

userReportParams.json

{
    "user": "lgreen",
    "startTime": 1447690682,
    "endTime": 1447699651,
    "maxResults": 2
}

Requesting an authentication token

Here is an example of a curl command requesting an authentication token. Requesting an authentication token does not require a request body, but does require an HCP Anywhere username and password. In this example the HCP Anywhere username is lgreen and the password is mypassword:

Curl command request

curl -kiu "lgreen:mypassword" -d "grant_type=urn:hds:oauth:negotiate-client&version=2.1.0&uniqueId=myUniqueID&clientNickname=myName" -H "X-HCPAW-MANAGEMENT-API-VERSION:2.1.0" https://api.anywhere.example.com:8000/login/oauth

The response you receive if credentials are valid

{
"access_token":""access_token":

"token_type":"Bearer",

Request with cURL command line

Here is an example of a curl command requesting the file modifications audit report. This command uses the userReportParams.json file you created earlier and uses the access token you received when you requested an authentication token. In this example the authentication token ends with /u003d. You need to replace /u003d with an equal sign (=) before using the token in your curl command request.

Curl command request

curl -kiX POST -T userReportParams.json -H "Accept: application/json" -H "Authorization: Bearer bH30R3pFE5w4LD25OJmjZS/7jS/ZF6gPVSkNWVsjQNinFxcOUeXEHcX26eACrvjH5SUX29C8DVygdGaH2n5hnlzlkxVpqxgdc4ai+9w=" -H "X-HCPAW-MANAGEMENT-API-VERSION: 2.1.0" -H "X-HCPAW-DEVICE-ID:myUniqueID" "https://api.anywhere.example.com:8000/mapi/report/audit/user/activity/file/modifications"

Request headers

POST /mapi/report/audit/user/activity/file/modifications/1.1
Authorization: bH30R3pFE5w4LD25OJmjZS/7jS/ZF6gPVSkNWVsjQNinFxcOUeXEHcX26eACrvjH5SUX29C8DVygdGaH2n5hnlzlkxVpqxgdc4ai+9w\u003d
Content-Type: application/json;charset=UTF-8
Content-Length: 1032

Creating a paging token input parameter

Here is an example of how to edit the userReportParams.json file to request an additional page of the audit report. This example uses a shortened page token. The page token in an actual report is longer.

userReportParams.json

{
"pageToken": "+pwZ104r7ERfS3vPnSI/qw9RcJhm4d/OqQEOJvjVTe0Q1\nf8R59hdu6OJu8Q\u003d"
}

Curl Command for requesting a subsequent page

Here is an example of how to request an additional page of the audit report. Notice that the curl command does not change, only the request body.

Curl command request

curl -kiX POST -T userReportParams.json -H "Accept: application/json" -H "Authorization: Bearer ya29.qwHWIcw1FLhJh4-Y84FP_mp-PqaTgS5vHCNENuAB42qQ_4C5QuO4tmvTFE-XeLbYtDFhDGN65dA6yg" -H "X-HCPAW-MANAGEMENT-API-VERSION: 2.1.0" -H "X-HCPAW-DEVICE-ID:myUniqueID" "https://api.anywhere.example.com:8000/mapi/report/audit/user/activity/file/modifications"
