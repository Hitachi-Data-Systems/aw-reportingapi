Administrative report APIs

If you have the Administrator role, you can use the HCP Anywhere Reporting API to generate administrative reports. You can run a report against the entire system or all the users from a specific profile. The table below lists the resources for which you generate an administrative report and describes the report generated for each resource.

Method

Returning Information

 Scope Notes
…/mapi/report/admin/user/profileOverrides
POST

Returns all users in the system or specified profile that have settings that have been overridden, and reports the overridden settings

 System or Profile  
…/mapi/report/admin/user/authProviderAccess
POST Reports the external state in the authentication provider of users in the system or specified profile(s). System or Profile

The possible values that can be returned for a user's external state are DELETED or DISABLED.

Only users with one of these external states are returned. Active users are not included in the report output.
…/mapi/report/admin/user/access
POST

Reports the HCP Anywhere access status of all users in the HCP Anywhere system or specified profile(s).

 System or Profile

A user may not be able to access AW if:

Their account is disabled in AW

They are not a member of a profile that has File Sync and Share access

The noDataAccess field may not be accurate until the user service runs, which by default occurs once per day.
…/mapi/report/admin/user/orphaned
POST Reports all users in the system that are not a member of any profile. System The list of reported users may not be accurate until the user service runs, which by default once per day.
…/mapi/report/admin/user/lastAccess
POST Reports the last access time for a user in the system or specified profile(s). System or Profile This command takes an optional inactivityTimeSeconds parameter. Only users that have been inactive for longer than the provided time are returned. If the inactivityTimeSeconds parameter is not supplied, then all users in the specified profile(s) are returned in the report output.
…/mapi/report/admin/user/storage
POST Reports the amount of data stored in the HCP Anywhere system by users in the specified profile(s). Profile  
…/mapi/report/admin/system/storage
POST Reports the amount of data stored in the HCP Anywhere system. System

The report is never paged.

There are no input parameters for this report.
.../mapi/report/admin/user/highQuotaUsage
POST Reports all users in the system or specified profile(s) near or above their allocated quota. System or Profile For more information on configuring quota, see Creating a profile.
…/mapi/report/admin/user/devices
POST Reports all registered HCP Anywhere devices of users in the system or specified profile(s). System or Profile  
.../mapi/report/admin/teamFolders
POST Reports all existing Team Folders in the system. System For each Team Folder in the system, the following info will be given: Label, contact, used quota, and allocated quota.
