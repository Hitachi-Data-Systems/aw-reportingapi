File sync and share settings

For each profile that has been created, you can adjust the users' file sync and share settings. The file sync and share settings include:

•How the users in the profile can share

•Who the users in the profile can share with

•How many and which HCP Anywhere applications the users in the profile can install

•The NAS shares that the users in the profile can access in HCP Anywhere

To edit the file sync and share settings for an existing profile:

1.In the top-level menu in the Management Console, mouse over Access.

2.In the menu, click on Profiles.

3.In the list of profiles, click on the profile you want.

4.In the profile, click on the File Sync and Share tab.

5.To enable or disable file sync and share, select or deselect the Enable file sync and share option, as applicable.

6.To configure file sync and share settings, make the changes you want on the Core, Devices, Sharing, Clients, Mobilization, and Data Exclusion tabs:

Core

•Quota — In this field, enter the amount of space each user has for adding files to HCP Anywhere.

Devices

Note: If the File Sync and Share API is an enabled device type in the profile, all applications connected to a user's HCP Anywhere account through the File Sync and Share API count as a single device. For example, if a user has three File Sync and Share API applications connected to HCP Anywhere, those three applications collectively count as one device registered to the user's HCP Anywhere.
List of device types

•Windows desktop — When selected, users in the profile can register the HCP Anywhere Windows desktop application with the system.

•Mac — When selected, users in the profile can register the HCP Anywhere Mac desktop application with the system.

•iOS — When selected, users in the profile can register the HCP Anywhere iOS application with the system.

•Android —When selected, users in the profile can register the HCP Anywhere Android application with the system.

•Windows Outlook Add-in — When selected, users in the profile can register the HCP Anywhere Windows Outlook Add-in with the system.

Note: For more information on any of the above applications, see theHCP Anywhere User Help.

•FSS API — When selected, users in the profile can authenticate with the HCP Anywhere File Sync and Share API or applications built using the File Sync and Share API. The File Sync and Share API is an HTTP API for integrating custom and third-party applications with HCP Anywhere. For more information on the File Sync and Share API, see the HCP Anywhere File Sync and Share API Reference Guide, which is available in the Management Console. For information on accessing the HCP Anywhere documentation, see Accessing documentation.

•HCP Anywhere Edge registration — When enabled, users in the profile can register HCP Anywhere Edge devices with the system. For more information on HCP Anywhere Edge, see Install and manage Edge.

•Data Management API — When enabled, users in the profile can authenticate with the HCP Anywhere Data Management API or applications built using the Data Management API. The Data Management API is an internal API that the HCP Anywhere service team uses to make custom tools for customers, such as a tool for moving data into HCP Anywhere. For more information, contact your HCP Anywhere service provider.

Automatic Deregistration

In this section, you can configure the deregistration settings for the users' HCP Anywhere applications.

•Automatically deregister after N days of inactivity — The amount of time an HCP Anywhere application can be inactive before the device is disconnected from HCP Anywhere. You can enter any integer that is less than the Archive File Events setting. See Archiving file event records for information on that setting.

When an application is deregistered from HCP Anywhere, the HCP Anywhere server stops synchronizing files with the application.

•Delete all files when an inactive desktop device is deregistered by the system — When selected, the files in a user's HCP Anywhere folder are deleted from their computer if their desktop application is deregistered by the system. This deletes the files on the user's computer, not in HCP Anywhere.

By default, when a mobile application is deregistered, all the user's files are deleted from the application.

Note: You can also deregister the applications of individual users instead of deregistering applications on a profile level. For information on doing this, see Deregistering user devices.
Authentication Token Expiration

In this section, you can configure the amount of time that the authentication token lasts for each type of device. When a user installs and registers an HCP Anywhere application on a device, the device is registered with the HCP Anywhere system and given an authentication token. When the token expires, the user needs to reenter his or her password to continue using the application.

•Desktop devices — The amount of time that the authentication token remains active for the HCP Anywhere desktop applications or Outlook Add-ins. You can choose any time between 1 day and 50 years. The default is 7 days.

•Mobile devices — The amount of time that the authentication token remains active for the HCP Anywhere mobile applications. You can choose any time between 1 day and 50 years. The default is 7 days.

•FSS API — The amount of time that the authentication token remains active for applications built using the File Sync and Share API. You can choose any time between 1 day and 50 years. The default is 7 days.

•Data Management and Edge API — The amount of time that the authentication token remains active for Edge applications or applications built using the Data Management API . You can choose any time between 1 day and 50 years. The default is 7 days.

Note: For more information on how Edge applications use their authentication token, see Enable Edge in a profile.
Sharing

•Enable read link sharing — When selected, users in the profile are allowed to share links internally with read access. A user who receives a link with read access can view and download the file or folder that was shared.

•Enable public read link sharing — When selected, users in the profile can share links publicly with read access. If a link is shared publicly, it can be accessed by anyone, even those who are not HCP Anywhere users.

•Enable upload link sharing — When selected, users in the profile can share links internally with upload access. Upload access only applies to folders. A user who receives a link with upload access can upload files to the folder that was shared. However, if the link is not also set with read access, the user cannot view or download any of the files in the folder.

•Enable public upload link sharing — When selected, users in the profile can share links publicly with upload access. Upload access only applies to folders. A user who receives a link with upload access can upload files to the folder that was shared. If a link is shared publicly, it can be accessed by anyone, even those who are not HCP Anywhere users.

•Allow links without expiration — When selected, users can share links that never expire.

•Maximum days to share — The maximum amount of time that a link can remain active. You can enter any integer from 1 to 18,262 days (1 day through 50 years). If Allow links without expiration is enabled, this setting does not apply.

•Default days to share — The default amount of time that links remain active. A user can change this value when sharing a link.

•Require access code on public links — When selected, access codes are enforced on public links. Access codes offer additional link security. Links generated with an access code can only be accessed by a person with both the link and the code.

•Require access code on all links — When selected, access codes are enforced on all links (public or internal). Access codes offer additional link security. Links generated with an access code can only be accessed by a person with both the link and the code.

Note: Link sharing can be enabled or disabled for the entire system. For information on doing this, see Configuring link sharing for the system.
Team Folder Creation

In this section, you can enable team folder creation. Team folders are folders that users can share with other HCP Anywhere users and that use system storage instead of personal storage.

•Enable Team Folders — When selected, users in the profile can create and share team folders.

•Threshold for requests — The maximum amount of storage space permitted for each team folder. The user who creates a team folder sets the storage space for the folder. If a user sets the storage space of a team folder over this limit, a request for the additional space is sent to you for approval. For information on answering these requests, see Accept and decline team folder storage requests.

Note: Unlike links, you can work with the team folders that users create. On the Team Folders page in the Management Console, you can view all the existing team folders in the system, answer team folder storage requests, invite users to manage team folders, and delete team folders. For information on doing these things, see Team folders.
Limit Sharing

In this section, you can limit sharing by preventing different profiles from sharing with one another.

•Limit folder and link sharing — When selected, members of the current profile can only initiate sharing with those who are part of one of the profiles in the Added Profiles section. This applies for both link sharing and folder sharing.

•Available Profiles— This table contains all the profiles in the HCP Anywhere system. Click on the plus sign ( ) to add a profile to the Allow Sharing with These Profiles section.

•Allow Sharing with These Profiles — This table contains the profiles that the current profile can share with when limit sharing is enabled.

Important:  

•Since users do not need to authenticate to access a public link, limiting sharing here does not prevent users from sending public links. If you want complete control over who your users can share with, enable limit sharing and disable public links.

•If you want to prevent users from sharing with anyone, enable limit sharing, make sure the Allow Sharing with These Profiles section does not contain any profiles, and disable public link sharing.

•Limit sharing is one directional. For example, if you limit sharing in profile A so that profile A cannot initiate sharing with profile B, profile B can still initiate sharing with profile A. To completely limit sharing between the two profiles, you need to limit sharing in both profiles.

Clients
Desktop Clients

In this section, you can configure several desktop application settings for your users. Users can also adjust these same settings, if you enable it, in their own desktop applications on the Settings page.

•All desktop settings are user configurable — When checked, the Desktop Clients settings that you configure:

oAre applied to every desktop application that is registered with the system after these settings are configured, but are not applied to any existing desktop applications already in the system

oCan be changed by users in their own desktop applications on the Settings page

When unchecked, the Desktop Clients settings that you configure:

oAre applied and then enforced on all new and existing desktop applications

oCannot be changed by users

•Enable alerts — When checked, alerts about the desktop application appear on the desktop application icon () for Windows and in the Mac notification center for Mac. Alerts give users information about their desktop applications, including when file conflicts occur, a desktop application upgrade is available, or the synchronization status of a file changes.

•Start automatically on login — When checked, the desktop application automatically starts whenever a user logs into their computer. The desktop application needs to be running in order to synchronize files to and from the HCP Anywhere system.

•Allow syncing of locked files (Windows only) — When checked, the Windows desktop application automatically attempts to synchronize any locked files it detects in a user's HCP Anywhere folder. Files can be locked by other programs or Windows itself. For example, a file can become locked if open in another program, such as Microsoft Word or PowerPoint.

Outlook Add-in Clients

In this section, you can configure several Outlook Add-in settings for your users. Users can also adjust these same settings, if you enable it, in their own Outlook Add-ins on the Settings page.

•Convert attachments to links — When users attach a file to an email, HCP Anywhere can be configured to automatically convert the file into an HCP Anywhere link.

oAlways (using default location and link settings) — When checked, attachments are always converted into links with the user's default link settings and upload location. In the Outlook Add-in, a user's default link settings are configured on the Link Settings tab and the default upload location is configured on the General tab. On this Management Console page, you can also configure a default upload location for the users in the profile.

oAsk for location and link settings every time — When checked, users are prompted to convert their attachments into links. In the prompt, users can configure their link settings and select an upload location or choose to leave the file as an attachment.

oNever — When checked, users are never prompted to convert attachments into links.

oConvert attachments setting is user configurable — When checked, the Convert attachments to links settings that you configure:

–Are applied to every Outlook Add-in that is registered with the system after these settings are configured, but are not applied to any existing Outlook Add-ins already in the system

–Can be changed by users in their own Outlook Add-ins on the Settings page

When unchecked, the Convert attachments to links settings that you configure:

–Are applied and then enforced on all new and existing Outlook Add-ins

–Cannot be changed by users

•Default upload location — When a user converts an attachment into a link using the add-in, the file is uploaded to that user's HCP Anywhere account.

oIn the field, enter the default upload location for attachments that are converted to links.

oDefault upload setting is user configurable — When checked, the Default upload location setting that you configure:

–Is applied to every Outlook Add-in that is registered with the system after these settings are configured, but is not applied to any existing Outlook Add-ins already in the system

–Can be changed by users in their own Outlook Add-ins on the Settings page

When unchecked, the Default upload location setting that you configure:

–Is applied and then enforced on all new and existing Outlook Add-ins

–Cannot be changed by users

•Collision resolution — A collision occurs when a file is uploaded to HCP Anywhere with the same name as another file already at that location.

oAutomatically rename — When checked, the add-in automatically renames the new file (by appending a number to the end of the file name) when a collision occurs.

oAsk me every time — When checked, the add-in prompts users to rename the new file or overwrite the existing one when a collision occurs.

oAutomatically overwrite — When checked, the add-in automatically overwrites the existing file when a collision occurs. In this case, the new file becomes a new version of the existing file.

oCollision resolution setting is user configurable — When checked, the Collision Resolution settings that you configure:

–Are applied to every Outlook Add-in that is registered with the system after these settings are configured, but are not applied to any existing Outlook Add-ins already in the system

–Can be changed by users in their own Outlook Add-ins on the Settings page

When unchecked, the Collision Resolution settings that you configure:

–Are applied and then enforced on all new and existing Outlook Add-ins

–Cannot be changed by users

•Advanced — The advanced Outlook Add-in setting applies to the Convert attachments to links setting:

oOnly convert attachments greater than: — When checked, files smaller than the size you specify in the field below this option are never automatically converted to links.

oAdvanced settings are user configurable — When checked, the Advanced setting that you configure:

–Is applied to every Outlook Add-in that is registered with the system after these settings are configured, but is not applied to any existing Outlook Add-ins already in the system

–Can be changed by users in their own Outlook Add-ins on the Settings page

When unchecked, the Advanced setting that you configure:

–Is applied and then enforced on all new and existing Outlook Add-ins

–Cannot be changed by users

Mobilization

In this section, you can configure the network shares that the users in the profile can access in their HCP Anywhere accounts. Each network share appears as a folder in HCP Anywhere.

Notes:  

•If your networks shares are not available on this page, it is because they have not been added to the HCP Anywhere system. See Working with NAS servers for information on doing this.

•If SAML authentication is enabled for the profile, network shares cannot be added to the profile.

To make a network share available to the users in the profile:

a.In the Server field, select a NAS server.

b.From the list of shares that appears, select the shares you want.

c.Click on the right arrow button (  ).

d.Click on Save Settings.

Data Exclusion

•Exclude these file extensions — When selected, users in the profile cannot upload files to HCP Anywhere with the extensions are in the file extension field.

•In the file extension field, enter the file extensions that you want users to be prevented from uploading to HCP Anywhere. For example, if you enter png into the file extensions field, users cannot upload files with the png extension.

Note: Do not include periods (".") in the list of file extensions. If a period is included, users are still able to upload files with the specified extension.
