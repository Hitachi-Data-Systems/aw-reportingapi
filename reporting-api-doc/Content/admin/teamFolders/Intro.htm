Team folders

Team folders can be created in two ways:

•By users in the User Portal, if you enable it

•By an HCP Anywhere Edge application, if you have one installed

Team folders created by users

Team folders allow users to create shared folders that don't use any of their personal HCP Anywhere storage space. Instead, the files inside a team folder use storage space from the system that you configure.

Once you enable team folder sharing, users can create and share team folders in the User Portal. To share a team folder, users send invitations. When a user accepts a team folder invitation, they gain access to the folder that was shared. The folder members can then begin collaborating on the files in the folder together.

Unlike links, you can work with the team folders that users create. On the team folders page, you can view all the existing team folders, answer team folder storage requests, invite managers to team folders, and delete team folders as needed.

Team folders created by an Edge application

Team folders can also be created by an Edge application, if you have one installed. Edge is an HCP Anywhere application that moves data from a Windows Servers (network-attached storage) to HCP Anywhere (cloud storage). For more information, see Install and manage Edge.

Just like team folders created by users, team folders created by Edge appear on the Team Folders page. However, more often, you work with team folders created by Edge on the Device Management page instead of the Team Folders page. The Device Management page groups team folders by the Edge application that created them.

Team folders created by Edge do not:

•Use the storage quota set on the Team Folders > Configuration page. This quota is only for team folders created by users.

•Appear as abandoned (no users) or orphaned (no managers) on the Team Folders > Folder Management page. While you should be alerted when a team folder created by a user is orphaned or abandoned, in an Edge setup, abandoned or orphaned team folders can be common.

•Do not get reclaimed (deleted) when the folder is abandoned for a certain amount of time. The amount of time a team folder created by a user needs to be abandoned before it is reclaimed by the system is set one the team folders > Configuration page.
