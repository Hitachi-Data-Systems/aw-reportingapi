HCP Anywhere Reporting API Reference Guide

Step 1: Enable Edge in a profile

To register an Edge device, you need to be part of a profile that has the Edge device type enabled. This device type helps you to enforce who can and cannot register an Edge device. By default, the Edge device type is enabled in each profile.

For more information on profiles and how to create one, see Creating a profile.

How Edge is tied to its user and that user's profiles

Each Edge device is tied to the last user who registered or authenticated the device and the profiles that user is in.

To ensure that your Edge devices run continuously:

1.Create special AD users specifically for registering your Edge devices

2.Create a special Edge profile for these AD users

3.Use only the AD users in the Edge profile to register Edge devices with the HCP Anywhere system

4.Do not delete the users in AD, delete the profile those users are in, remove those users from the profile they're in, or disable the Edge device type in the profile those users are in

To enable the Edge device type in an existing profile:

1.In the top-level menu in the Management Console, mouse over Access.

2.In the menu, click on Profiles.

3.In the list of profiles, click on the profile you want.

4.In the profile, click on the File Sync and Share tab.

5.On the left side of the profile, click on Devices.

6.In the Device types allowed section, select HCP Anywhere Edge.

7.Optionally, in the Authentication Token Expiration section, change the expiration time for the Data Management and Edge API. The default expiration time is one week.

Note: Edge uses a two-token authentication method to ensure that it's secure while still able to run continuously without requiring a user to periodically reauthenticate. The two-token method uses:

• An access token — This token is used to access data. The expiration time you set in the profile applies to the authentication token. Ideally, the access token is short lived and refreshed frequently since it is sent to the HCP Anywhere system with each request.

•A refresh token — This token is used to acquire a new authentication token from the HCP Anywhere system once the old authentication token expires.  The refresh token is permanent.

8.Click on Save Settings.
