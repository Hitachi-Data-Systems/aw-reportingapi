Creating a request body

For most report requests, you need to create a JSON request body. The parameters you place in the request body determine:

• The users, system, or profile to report on .

•The size of the report .

If a report is larger than the default maxResults, which is 100 entries, or the maxResults you specify in your response body, the report returns the maxResults and a page token. In order to get the next page of the report, you need to replace the request body with the page token. For more information on paging, see Requesting an initial report and paging.

Here is an example request body stored in a file named userReportParams.json. This is the request body format you would use for a file modifications audit report of a single user. The startTime for this report is Mon, 16 Nov 2015 16:18:02 GMT and the endTime is Mon, 16 Nov 2015 18:47:31 GMT. The times have been converted respectively to the Unix Timestamp Conversion (UTC) times 1447690682 and 1447699651. Note that these UTC times are in milliseconds. Similarly, any start and end times you enter need to be converted to UTC times in milliseconds.

Example request body

userReportParams.json

{
    "user": "lgreen",
    "startTime": 1447690682000,
    "endTime": 1447699651000,
    "maxResults": 20
}

.../mapi/report/admin/user/lastAccess
JSON Type Description
scope String

Required. This parameter determines whether you are addressing the system or a particular profile. Valid values:

•SYSTEM

•PROFILE

profile String The name of a profile. This parameter is not allowed unless the scope is PROFILE, in which case the profile parameter is required.
maxResults Integer Optional. The maximum number of entries to return. Default is 100. The minimum value is 1. The maximum value is 5000.
inactivityTimeSeconds Integer

Optional. The user inactivity time in seconds. If this parameter is provided, only users that have been inactive for at least as long as the specified time are returned. The default time is 0 seconds.

A user becomes inactive if their clients have not contacted the server and the user hasn't logged into the user portal and the user hasn't made a MAPI request in the time specified by this input parameter.
.../mapi/report/admin/user/storage
JSON Type Description
Scope String

Whether you are addressing the system or a particular profile. Valid values:

•SYSTEM

•PROFILE

profile String The name of a profile. This parameter is not allowed unless the scope is PROFILE, in which case the profile parameter is required.
maxResults Integer Optional. The maximum number of entries to return. Default is 100. The minimum value is 1. The maximum value is 5000.
includeVersions Boolean Optional. The default value is false. If this parameter is true, then the total storage for all of the user's versions is returned as well as the total storage for active versions.
quotaPercent Integer Optional. The default value is 0. If this parameter is provided, then only users with active versions that consume quotaPercent or more of their quota are returned in this report. If includeVersions is true then even if quotaPercent is provided, it's ignored and all users are be returned.
.../mapi/report/admin/system/storage


This resource does not use a request body.
