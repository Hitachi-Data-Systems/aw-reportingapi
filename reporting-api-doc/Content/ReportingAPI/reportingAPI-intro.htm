About the Reporting API

HCP Anywhere offers a Reporting API. The Reporting API can be used to generate reports, so that you can track:

• User activity

•Profile activity

•System activity

You can use the Reporting API to generate two type of reports:

•Administrative reports — Returns information on user configuration and system resource usage. You need to have the Administrator role to generate an administrative report.

• Audit reports — Returns information on your HCP Anywhere users' file sync and share activity. You need to have the Audit role to generate an audit report. You can also audit users' activity and filesystems in the Management Console. See Auditing registered users for information on doing this.

Each report is generated by following this general workflow:

1.Create an authentication token — You create an authentication token by submitting your HCP Anywhere username and password to the authorization endpoint, port 8000 or port 7999.

2.Create a request body — You create a request body by specifying the required input parameters for the type of report you want to generate.

3.Request the report — You request an administrative or audit report by sending your authentication token and request body to the target URL.

4.Optionally, page through the report — If the report is more than one page, you request additional pages of the report by replacing your request body with the paging token you receive with your report, and then resending the request.
